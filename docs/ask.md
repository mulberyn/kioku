# 学习过程中的问题记录

> 个人学习过程中遇到的问题，向大模型询问的内容记录。

## i18n 相关

我在实现个人博客的 i18n 功能：

1. 以 tag 为例，用户创建文章（默认中文）与后续新增英文翻译时，tag 在不同语言首次出现与命名的流程应如何设计？
2. 使用 tag 查找文章时，SEO 友好的 URL 结构应该怎样设计？如果某个 tag 在目标语言下没有对应文章，前后端应该如何处理？
3. 技术栈为 Go + Gin 后端、NuxtJS 前端，标准的 i18n 处理流程是什么？性能/速度优化应从哪些方面考虑？
4. 什么是“预计算计数（precomputed counts）”优化？它解决了什么问题？
5. 预计算计数在多语言（不止 zh/en，多语种）场景下，数据模型与查询逻辑应如何设计并优化？
6. 对于 tag、category、series、content（与文章一对一）、summary（与文章一对一）应如何建模？数据库表结构如何设计？
7. 术语上为什么更推荐使用 locale 而不是 lang？
